# RustPTA Lane

Use this lane when turning Rust concurrency into a Petri-net abstraction.

## Best Fit

- thread coordination,
- lock or semaphore patterns,
- send/receive channels,
- resource acquisition and release.

## Model Output

- code-to-net mapping,
- concurrency entities,
- blocking or waiting places,
- resource lifecycle transitions,
- abstraction notes.

## Keep in Mind

- state clearly what the abstraction ignores,
- keep code semantics separate from model semantics,
- preserve the concurrency meaning of key operations.
