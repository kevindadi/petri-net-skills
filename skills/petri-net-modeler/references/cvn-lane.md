# CVN Lane

Use this lane when token identity or value matters.

## Best Fit

- multiple token types,
- data-dependent transitions,
- guard-based routing,
- typed resources or requests.

## Model Output

- token domains or classes,
- guard notes,
- value-sensitive arcs or transitions,
- explanation of what would be lost in a plain P/T reduction.

## Keep in Mind

- preserve semantic distinctions,
- avoid flattening too early,
- state when a simpler P/T model is only an approximation.
